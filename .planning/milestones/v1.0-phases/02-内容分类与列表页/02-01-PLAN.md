# 用户决策摘要
- 分类展示方式: 独立路由 /tech/ 和 /fiction/
- 列表布局: 卡片式（PaperMod 默认）
- 内容密度: 摘要预览模式（标题 + 日期 + 摘要 + 分类标签）
- 列表排序: 日期倒序

# 技术实现要点
- 使用 Hugo sections 机制（content/tech/ 和 content/fiction/）
- 每个 section 目录需创建 _index.md 定义标题和描述
- PaperMod 原生支持 section 列表页
- 首页使用 union 合并多 section 文章
- 导航菜单通过 hugo.yaml 的 menu 配置
</context>

<task type="auto">
  <name>Task 4: 创建首页模板合并多 Section</name>
  <files>layouts/index.html</files>
  <action>
创建 layouts/index.html 覆盖默认首页，合并 tech 和 fiction 两个 section 的文章：

```html
{{- define "main" -}}
<main class="post-list">
  {{ $tech := where site.RegularPages "Section" "tech" }}
  {{ $fiction := where site.RegularPages "Section" "fiction" }}
  {{ $all := union $tech $fiction }}
  {{ $sorted := $all.ByDate.Reverse }}
  {{ $paginator := .Paginate $sorted }}

  {{ range $paginator.Pages }}
    {{ .Render "post_entry" }}
  {{ end }}

  {{ template "partials/pagination.html" . }}
</main>
{{- end -}}
```

此模板：
1. 使用 `where` 筛选 tech 和 fiction 两个 section
2. 使用 `union` 合并两个集合
3. 使用 `ByDate.Reverse` 按日期倒序排序
4. 使用 `.Render "post_entry"` 渲染卡片式布局
5. 包含分页支持
