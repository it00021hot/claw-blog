<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: 在 GitHub 仓库中配置 Pages 和自定义域名</name>
  <action>
    用户需要访问 GitHub 仓库设置完成以下操作:
    1. 进入仓库 Settings → Pages
    2. 在 Build and deployment 部分，Source 选择 "Deploy from a branch"
    3. Branch 选择 "gh-pages"，目录选择 "/(root)"
    4. 在 Custom domain 输入 gen-code.top
    5. 勾选 "Enforce HTTPS"（可能需要等待证书自动生成后才有此选项）

    注意: 需要先推送代码触发一次 Actions 运行后，才能看到 gh-pages 分支选项。
  </action>
  <how-to-verify>
    1. 推送代码到 main 分支: git push origin main
    2. 等待 Actions 运行完成
    3. 访问仓库 Settings → Pages，确认 Source 显示 "gh-pages"
    4. 确认 Custom domain 显示 "gen-code.top"
  </how-to-verify>
  <done>GitHub Pages 配置完成，Custom domain 设置为 gen-code.top</done>
</task>

<task type="checkpoint:human-action" gate="blocking">

  <action>
    用户需要登录 Cloudflare 控制台完成以下操作:
    1. 登录 Cloudflare，选择 gen-code.top 域名
    2. 进入 DNS 设置
    3. 添加以下记录:

    | Type | Name | Content | Proxy Status |
    |------|------|---------|--------------|
    | CNAME | www | it00021hot.github.io | Proxied |
    | A | @ | 185.199.108.153 | Proxied |
    | A | @ | 185.199.109.153 | Proxied |
    | A | @ | 185.199.110.153 | Proxied |
    | A | @ | 185.199.111.153 | Proxied |

    注意: A 记录的四 IP 是 GitHub Pages 的官方 IP 地址。
  </action>
  <how-to-verify>
    1. 登录 Cloudflare 控制台
    2. 进入 gen-code.top 的 DNS 设置
    3. 确认 CNAME 和 A 记录已添加且状态为 "Proxied"
  </how-to-verify>
  <done>Cloudflare DNS 记录配置完成</done>
</task>
